第2回　デザインについて

テーマ

Bootstrapを使用して時間を少ない時間でそこそこのホームページを作成する

やったこと

Bootstrapを使用してホームページを作成し、git pagesで公開

前提

  • あまり時間をかけない
  • HTML、CSSの初歩的なことは知っている

Bootstrapとは？

世界で最も人気のあるCSSフレームワーク
レスポンシブデザインにも対応しており、手軽におしゃれなデザインを実装可能

詳しくはこちらをご覧ください
https://getbootstrap.com/

なぜBootstrapを使ったのか？

  • 簡単にレスポンシブデザイン
  • 手軽にそこそこのデザインを作成
  • githubのスター数も多く、情報が豊富

git pagesとは？

HTML、CSS、JavaScriptファイルで作成された静的サイトなウェブサイトを公開できるサイトホスティングサービス

詳しくはこちらをご覧ください
https://help.github.com/ja/github/working-with-github-pages/about-github-pages

何をした？

  1. githubでリポジトリ作成
  2. 以下のディレクトリ、ファイルを作成し、1で作成したリポジトリへpush
    # ディレクトリ　start_hp
    # ファイル　start_hp/index.html
    <!DOCTYPE html>
    <html lang="en">
    <head>
    <meta charset="UTF-8">
    <title>Document</title>
    </head>
    <body>
    hello
    </body>
    </html>

  3. 作成したリポジトリのgithub画面で
    Settings → GitHub Pages → Source
    プルダウンのmaster branch を選択し、Save ボタンをクリック

    以下の形式でURLが作成される
    https://リポジトリ名.github.io/ディレクトリ名/.
  4. 作成されたURLへアクセス
    →　アクセス成功し、helloの文字が表示されている（公開されている）
  5. 以下のサイトを参考にhtml、CSSを使用し、index.htmlを修正後、push
    https://getbootstrap.com/
    https://skillhub.jp/courses/168（とてもわかりやすかったです）
  6. 完成！
    https://puben216.github.io/start_hp/.

Bootstrap以外だと、どんなCSSフレームワークがある？

以下のサイトがとても参考になりました。 https://www.seleqt.net/programming/10-smallest-and-fastest-frontend-web-dev-frameworks/

  • デザインの定義方法をある程度統一できる
  • あまりデザインに詳しくなくても、ある程度のデザインを作成できた →Bootstrap読み込んで数行コードを書くだけで、それなりのデザインができた
